Data Flow

End-to-End Pipeline

Follow a record from raw ingestion through transformations, quality validation, and into production analytics.

1

Event Ingestion

Real-time events stream via Kafka or Debezium CDC, batch files land on GCS, APIs are polled on schedule. The ingestion layer normalizes all sources into a unified Bronze write path.

Kafka Debezium Cloud Storage REST APIs
2

Bronze Landing

Raw records land in Delta Lake or Iceberg tables with full fidelity. Every row is enriched with _ingestion_ts, _source, and _batch_id metadata. Tables are append-only — no updates, no deletes.

Delta Lake 3.x Apache Iceberg 1.5 ACID
3

Silver Transformation

dbt models deduplicate records using ROW_NUMBER window functions over primary keys. SCD Type 2 MERGE statements track historical changes with valid_from / valid_to / is_current flags. Row hashing detects real changes vs. phantom updates.

dbt-spark PySpark 3.5 SCD Type 2 SHA-256 hash
4

Quality Gates

Great Expectations suites validate row counts, null rates, referential integrity, and statistical distributions. Failed expectations block the pipeline and page the on-call engineer. Passing suites emit quality scores to DataPlex.

Great Expectations GCP DataPlex Alerting
5

Gold Aggregation

Validated Silver data feeds star-schema Gold tables in BigQuery. Pre-computed aggregates, KPI materialized views, and partition-pruned query patterns deliver sub-second BI responses. Looker, Tableau, and Superset connect directly.

BigQuery Star Schema Materialized Views
6

GenAI Semantic Layer

Vertex AI embeddings index all Gold table metadata, column descriptions, and sample values. LangChain RAG translates natural-language queries into SQL, executes via Trino, and explains results in plain English via a FastAPI endpoint.

Vertex AI LangChain Trino FastAPI

Every component is designed for real-world scale, reliability, and cost-efficiency.

⚡

ACID Transactions

Full serializable isolation across all medallion layers. Concurrent writers never corrupt data — Delta Lake's optimistic concurrency and Iceberg's snapshot isolation handle it automatically.

Delta Lake Iceberg Serializable

Time Travel Queries

Query any table as of any past timestamp or version number. Invaluable for auditing, debugging, ML feature backfill, and recovering from bad writes. Zero additional storage overhead.

Schema Evolution

Add, rename, or reorder columns without rewriting historical data or breaking existing readers. Delta Lake and Iceberg both support backward-compatible schema changes with full metadata tracking.

src/lakehouse_manager.py 
# Ingest raw events to Bronze — append-only, full fidelity
async def ingest_to_bronze(
    self,
    df: DataFrame,
    table: LakehouseTable,
    source_system: str,
) -> dict:
    enriched = df.withColumn("_ingestion_ts", F.current_timestamp())
                 .withColumn("_source", F.lit(source_system))
                 .withColumn("_batch_id", F.lit(str(uuid4())))

    gcs_path = f"gs://{self.config.gcs_bucket}/bronze/{table.name}"

    if table.table_format == TableFormat.DELTA:
        enriched.write.format("delta")              \
            .mode("append")                             \
            .partitionBy("_ingestion_date")             \
            .save(gcs_path)

    elif table.table_format == TableFormat.ICEBERG:
        enriched.writeTo(f"iceberg.bronze.{table.name}") \
            .using("iceberg")                            \
            .partitionedBy("days(_ingestion_ts)")         \
            .append()

    return {
        "rows_written": enriched.count(),
        "gcs_path": gcs_path,
        "format": table.table_format.value,
    }
src/silver/transformer.py 
# SCD Type 2 MERGE — track full change history
def _build_scd2_merge(self, table: LakehouseTable) -> str:
    pk_join = " AND ".join(
        f"t.{k} = s.{k}" for k in table.primary_keys
    )
    hash_cols = ", ".join(table.primary_keys)

    return f"""
MERGE INTO silver.{table.name} t
USING (
    SELECT *,
           SHA2(CONCAT_WS('|', {hash_cols}), 256) AS _row_hash
    FROM   bronze_{table.name}_deduped
) s ON {pk_join}
   AND t.is_current = TRUE

WHEN MATCHED AND t._row_hash != s._row_hash THEN
    UPDATE SET t.valid_to    = current_timestamp(),
               t.is_current  = FALSE

WHEN NOT MATCHED THEN
    INSERT (*, valid_from, valid_to, is_current)
    VALUES (s.*, current_timestamp(), NULL, TRUE)
"""
src/gold/aggregator.py 
# Build Gold aggregation table in BigQuery
def build_gold_table(
    self,
    spark: SparkSession,
    source_query: str,
    table: LakehouseTable,
) -> None:
    df = spark.sql(source_query)

    # Write to BigQuery with partition and cluster config
    df.write.format("bigquery")          \
        .option("table", f"""{self.config.project}.
                {self.config.bq_dataset}.
                {table.name}""")           \
        .option("partitionField",
                table.partition_cols[0])   \
        .option("clusteredFields",
                ",".join(table.cluster_cols)) \
        .option("createDisposition",
                "CREATE_IF_NEEDED")         \
        .mode("overwrite")                 \
        .save()

    logger.info(f"Gold table {table.name} refreshed")
src/semantic_search/query_engine.py 
# Translate natural language → SQL → human-readable answer
class NLQueryEngine:
    def __init__(self, trino_conn: TrinoConnection):
        self.llm = VertexAI(model_name="gemini-pro")
        self.embeddings = VertexAIEmbeddings(
            model_name="textembedding-gecko@003"
        )
        self.trino = trino_conn
        self.vector_store = Chroma(
            embedding_function=self.embeddings,
            persist_directory="/tmp/catalog_index",
        )

    async def query(self, question: str) -> dict:
        # 1. Retrieve relevant table schemas via semantic similarity
        docs = self.vector_store.similarity_search(question, k=5)
        context = "\n".join(d.page_content for d in docs)

        # 2. Generate SQL with chain-of-thought reasoning
        chain = LLMChain(llm=self.llm, prompt=SQL_GENERATION_PROMPT)
        sql   = await chain.arun(context=context, question=question)

        # 3. Execute against Trino, explain result
        rows   = self.trino.execute(sql)
        answer = await self.llm.apredict(
            EXPLAIN_PROMPT.format(question=question, rows=rows[:20])
        )
        return {"sql": sql, "rows": rows, "answer": answer}
dbt_project/models/silver/dim_customers.sql 
-- dbt Silver model: deduplicated, SCD2-tracked customer dimension
{{ config(
    materialized = 'incremental',
    unique_key    = 'customer_id',
    incremental_strategy = 'merge',
    on_schema_change = 'sync_all_columns',
    file_format  = 'delta',
) }}

WITH ranked AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY customer_id
               ORDER BY     _ingestion_ts DESC
           ) AS rn
    FROM  {{ source('bronze', 'customers') }}
    {% if is_incremental() %}
    WHERE _ingestion_ts > (SELECT MAX(valid_from) FROM {{ this }})
    {% endif %}
),
deduped AS (
    SELECT * FROM ranked WHERE rn = 1
)

SELECT
    customer_id,
    email,
    tier,
    region,
    SHA2(CONCAT_WS('|', email, tier, region), 256) AS _row_hash,
    current_timestamp() AS valid_from,
    NULL               AS valid_to,
    TRUE               AS is_current
FROM deduped
trino/federation_query.sql 
-- Cross-source federated query: Delta Lake + BigQuery + PostgreSQL
SELECT
    o.order_id,
    c.email                  AS customer_email,
    c.tier                   AS customer_tier,
    p.name                   AS product_name,
    p.category,
    SUM(o.amount)            AS total_revenue,
    AVG(r.rating)            AS avg_review_rating,
    COUNT(DISTINCT o.order_id) AS order_count
FROM
    -- Delta Lake table on GCS via Hive metastore
    delta.gold.orders       o
    -- BigQuery Gold dimension
    JOIN bigquery.analytics.dim_customers c
         ON o.customer_id = c.customer_id
         AND c.is_current  = TRUE
    -- PostgreSQL operational catalog
    JOIN postgresql.ops.products p
         ON o.product_id = p.id
    -- Iceberg review table on GCS
    LEFT JOIN iceberg.silver.reviews r
         ON o.order_id = r.order_id
WHERE
    o.order_date >= DATE_ADD('day', -30, CURRENT_DATE)
GROUP BY 1,2,3,4,5
ORDER BY total_revenue DESC
LIMIT 100

Format Decision Guide

Delta Lake vs Apache Iceberg

Both formats are supported. Choose based on your query engine ecosystem and operational requirements.

Feature Delta Lake 3.x Apache Iceberg 1.5 Winner
ACID Transactions ✓ Optimistic CC ✓ Snapshot Isolation Tie
Time Travel ✓ VERSION / TIMESTAMP ✓ Snapshot-based Tie
Schema Evolution ✓ Full support ✓ Full + type widening Iceberg (type widening)
Spark Integration ✓ Native (DataBricks) ✓ Full support Delta (tighter)
Trino / Presto ⚡ Good ✓ Native first-class Iceberg
Flink Integration ⚡ Limited ✓ Full streaming Iceberg
BigQuery Integration ✓ BQ Omni ✓ BQ External Tables Tie
Small File Problem ✓ OPTIMIZE + ZORDER ✓ Rewrite Data Files Tie
Partition Evolution ✗ Static partitions ✓ Hidden partitioning Iceberg
Branching / Tagging ✗ Not supported ✓ Branch + Tag API Iceberg
Ecosystem (2025) ✓ Databricks-driven ✓ Vendor-neutral Iceberg (broader)
Quickstart

Get Running in Minutes

From clone to first Bronze ingestion in under 5 minutes.

1

Clone & Install

Clone the repository and install all dependencies including PySpark, Delta Lake, and GCP libraries.

git clone https://github.com/jthiruveedula/data-lakehouse-modern-stack
cd data-lakehouse-modern-stack
pip install -r requirements.txt
2

Configure GCP

Set up your GCP project credentials and configure the lakehouse settings with your project ID and GCS bucket.

gcloud auth application-default login
export GCP_PROJECT="your-project-id"
export GCS_BUCKET="your-lakehouse-bucket"
3

Provision Infrastructure

Use Terraform to provision Dataproc, GKE for Trino, GCS buckets, and IAM roles in one command.

cd infrastructure/
terraform init
terraform apply -var="project=$GCP_PROJECT"
4

Run First Ingestion

Ingest your first dataset into the Bronze layer. The orchestrator handles partitioning, metadata enrichment, and table registration automatically.

from src import MedallionPipelineOrchestrator
orch = MedallionPipelineOrchestrator(config)
await orch.ingest_to_bronze(df, table, "source")
5

Run dbt Transformations

Execute dbt models to transform Bronze data through Silver deduplication and SCD2 into Gold aggregation tables.

cd dbt_project/
dbt deps
dbt run --select silver+
dbt test
6

Query with Natural Language

Start the semantic search API and ask data questions in plain English. The GenAI layer translates to SQL and returns answers.

uvicorn src.semantic_search.api:app --reload

curl -X POST /query \
-d '{"question": "Top 10 customers by revenue last month?"}'
